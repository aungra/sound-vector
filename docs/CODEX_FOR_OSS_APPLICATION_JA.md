# Codex for Open Source 応募メモ

応募ページ: https://openai.com/ja-JP/form/codex-for-oss/

このプログラムは、活発にメンテナンスされているOSSのメンテナーを支援するものです。フォームでは公開GitHubリポジトリ、メンテナー種別、対象理由、APIクレジットの利用計画、OpenAI組織IDが必要です。

## リポジトリ説明

Sound Vector Fitting Room は、音源を解析して編集可能なSVG図案に変換し、SVG内の保護ジオメトリから音声スケッチを復元するOSSプロトタイプです。音楽、印刷、アーカイブ、教育、クリエイティブコーディングで共有できる「可逆音響SVG」フォーマットを目指しています。

## 「このリポジトリが対象となる理由」500字以内案

音声をSVGへ変換し、SVGから音声スケッチを復元する可逆フォーマットのOSSです。音楽作品を画像・印刷物・展示物として扱いながら、`pcm_reversible_data` レイヤーに復元用ジオメトリを保持します。既存の音声可視化は一方向の画像化が中心ですが、本リポジトリはデザイナーがIllustratorで編集でき、同時に機械可読な音響データも残せる共通基盤を目指します。音楽アーカイブ、教育、生成デザイン、アクセシビリティにまたがる小さなエコシステムを作るため継続開発します。

## 「APIクレジットをどのように活用する予定ですか」500字以内案

CodexとAPIクレジットは、可逆SVGコーデックの分離、テスト生成、Issue/PR整理、SVGパーサー差分の検証、セキュリティ確認に使います。具体的には `audio -> svg`、`svg -> audio`、`svg inspect` のCLI化、音声特徴量と復元品質を比較するフィクスチャ作成、ブラウザ版とNode版の実装差分レビュー、Illustrator編集後の互換性検証を自動化します。最終的に、誰でも再現できる仕様書・テスト・サンプルSVGを整備します。

## 「他に伝えたいこと」500字以内案

現在は展示用プロトタイプからOSS基盤へ整理している段階です。応募時点でスター数やダウンロード数はまだ小さい可能性がありますが、音声、SVG、印刷、アーカイブをつなぐ可逆フォーマットとして明確な新規性があります。Codexを使い、単体HTMLの実験コードを保守可能なライブラリ、CLI、仕様書、テストへ分割し、外部コントリビューターが参加しやすいリポジトリへ育てます。
